## 1. sudo 입력하지 않기

Docker의 명령은 `docker <명령>` 형식으로, 항상 root 권한으로 실행해야 함.
docker 명령은 root 권한으로 실행해야 하기 때문에 일반 계정에서는 항상 sudo를 사용해야 함
But, 귀찮기 때문에 sudo를 입력하지 않는 방법이 있음

(1) root 계정으로 로그인하거나, `sudo su`을 사용해 root 계정으로 전환

```bash
$ sudo su
#
```

(2) 현재 계정을 docker 그룹에 포함시키기 (현재 계정에서 로그아웃한 뒤 다시 로그인)

```bash
$ sudo usermod -aG docker ${USER}
$ sudo systemctl restart docker
```

## 2. search : 이미지 검색

```bash
docker search ubuntu
```

- [Docker Hub](https://hub.docker.com)를 통해 이미지를 검색하는 명령어
- https://hub.docker.com/_/ubuntu/tags

---

## 3. pull : 이미지 받기

```bash
# docker pull <이미지 이름>:<태그>
docker pull ubuntu:latest
```

---

## 4. images : 이미지 목록 출력

```bash
# 모든 이미지 목록을 출력
docker images
```

---

## 5. run : 컨테이너 생성

```bash
# docker run <옵션> <이미지 이름> <실행할 파일>
docker run -i -t --name hello ubuntu /bin/bash
```

---

## 6. ps : 컨테이너 목록 확인

```bash
# 실행되고 있는 컨테이너만 출력
docker ps

# 정지된 컨테이너까지 모두 출력 (all 옵션)
docker ps -a
```

---

## 7. start : 컨테이너 시작

```bash
# docker start <컨테이너 이름 or 컨테이너 ID>
docker start hello
```

---

## 8. restart : 컨테이너 재시작

```bash
# docker restart <컨테이너 이름>
docker restart hello
```

---

## 9. attach : 컨테이너에 접속

```bash
# docker attach <컨테이너 이름>
docker attach hello
```

---

## 10. exec : 외부에서 컨테이너 명령 실행

```bash
# docker exec <컨테이너 이름> <명령> <매개 변수>
docker exec hello echo "Hello World"
```

---

## 11. stop : 컨테이너 정지

```bash
# docker stop <컨테이너 이름>
docker stop hello
```

---

## 11. rm : 컨테이너 삭제

```bash
# docker rm <컨테이너 이름>
docker rm hello
```

---

## 12. rmi : 이미지 삭제

```bash
# docker rmi <이미지 이름>:<태그>
docker rmi ubuntu:latest
```
